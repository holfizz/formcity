# Публикация через VS Code

## Быстрый способ

1. **Откройте панель Source Control**

   - Нажмите `Ctrl+Shift+G` (Windows/Linux)
   - Или `Cmd+Shift+G` (macOS)
   - Или кликните на иконку ветки слева

2. **Нажмите кнопку "Publish Branch"**

   - Она появится вверху панели Source Control

3. **Выберите тип репозитория**

   - Public (публичный) - виден всем
   - Private (приватный) - только для вас

4. **Подтвердите публикацию**
   - VS Code автоматически создаст репозиторий на GitHub
   - И загрузит все файлы

## Что будет загружено

✅ **Будет загружено:**

- Весь исходный код
- package.json и зависимости
- README и документация
- .env.example (без секретов)
- Тестовые данные (data.csv)

❌ **НЕ будет загружено (в .gitignore):**

- .env (с реальными токенами)
- node_modules/
- dist/
- Логи и временные файлы

## После публикации

1. **Проверьте репозиторий на GitHub**

   - Откройте ссылку, которую покажет VS Code
   - Убедитесь, что .env файла нет

2. **Добавьте описание репозитория**

   - На странице GitHub нажмите "Edit"
   - Добавьте описание и теги

3. **Настройте README**
   - Добавьте бейджи
   - Обновите инструкции по установке

## Важно!

Перед публикацией убедитесь, что:

- [ ] Файл .env в .gitignore
- [ ] В .env.example нет реальных токенов
- [ ] Удалены конфиденциальные данные
- [ ] Обновлен README с инструкциями

## Если что-то пошло не так

Если случайно загрузили .env с токенами:

1. **Немедленно смените все токены:**

   - Telegram Bot Token
   - OpenAI API Key

2. **Удалите файл из истории:**

   ```bash
   git filter-branch --force --index-filter \
     "git rm --cached --ignore-unmatch .env" \
     --prune-empty --tag-name-filter cat -- --all

   git push origin --force --all
   ```

3. **Или удалите репозиторий и создайте новый**
